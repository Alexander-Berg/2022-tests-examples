В этой папке содержатся файлы для воспроизведения ошибок от сервера.

Зачем мы это делаем:
При появлении новых походов в сеть в приложении есть необходимые к выполнению базовые проверки, с помощью которых избегаем появления непредвиденных проблем/падений в проде.

Как найти запрос, требующий подмены:
При проверке новой фичи может возникнуть ситуация, когда необходимая ручка еще не описана, но при этом известно при каких действиях она будет появляться.
В таких случаях достаточно подключить устройство к снифферу, закрыть лишние приложения, которые могут посылать запросы в сеть и в проверяемом приложении выполнить указанные действия.
При этом в сниффере моргнут выделением все ручки, в которых в этот момент пришли новые запросы


Содержимое папок:

- errors_maps - файлы для подмены ответов сервера в МЯК
    - 301, 404, 500 - очевидно
    - invalid_symbols - невалидные символы &rlm; &#8207; \200F U+200F Знак справа-налево ¡ &iexcl; &#161; . Чтобы charles не экранировал, добавлено Сontent-Type: application/x-binary
    - empty - пустой файл
    - empty_object - пустой объект в ответе
    - random_text - ответ с набором рандомных символов
- errors_metro - файлы для подмены ответов сервера в Метро
